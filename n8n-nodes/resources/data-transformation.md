---

## 1. Edit Fields (Set) 節點

這是工作流中最常用的資料處理節點，用於新增、修改或刪除項目中的欄位。

| 功能              | 說明                                                            |
| :---------------- | :-------------------------------------------------------------- |
| **新增/更新欄位** | 使用表達式 `{{ $json.field }}` 引用上游資料，或直接輸入固定值。 |
| **刪除欄位**      | 移除項目中不再需要的敏感資訊或冗餘資料。                        |
| **類型轉換**      | 支援將字串轉換為數值、布林值或對象，確保下游節點能正確解析。    |

## 2. Code 節點 (自定義腳本)

當內建節點無法滿足複雜的邏輯需求時，`Code` 節點提供最強大的擴充性。

- **JavaScript/Python 支援**：您可以選擇慣用的語言來處理資料。
- **資料處理模式**：
    - **JavaScript**：通常使用 `items.map()` 或 `for` 迴圈來迭代處理每一筆資料。
    - **Python**：使用標準的列表推導式或迴圈進行轉換。
- **外部庫**：支援在沙盒環境中使用特定的內建函式庫（如 `crypto` 或 `moment`）。

## 3. Aggregate 節點 (資料聚合)

將多個輸入項目合併為單一項目，常用於生成總結報告或批次操作。

- **聚合模式**：
    - **Sum/Average**：對數值欄位進行加總或計算平均值。
    - **Min/Max**：找出數組中的最小值或最大值。
    - **Array**：將所有項目的特定欄位收集到一個陣列中。
- **分組聚合 (Group By)**：可以根據特定的 Key（如 `category`）對資料進行分組後再聚合。

## 4. Filter 節點 (資料篩選)

根據特定條件過濾掉不符合要求的項目，保持工作流的資料純淨。

- **條件組合**：支援 AND/OR 邏輯組合多個篩選條件。
- **進階過濾**：除了基本的數值和字串比較，還支援「欄位是否存在」、「是否為空」等檢查。

## 5. Sort 節點 (排序)

對輸入項目進行重新排列，這對於生成有序列表或準備資料匯出至試算表非常有用。

- **多級排序**：可以先按 `date` 降序排列，再按 `id` 升序排列。
- **類型識別**：自動識別數值、字串和日期類型，確保排序結果符合預期。

## 6. Rename Keys 節點

當下游系統要求的欄位名稱與上游不一致時，此節點能快速完成對應。

- **批量重新命名**：一次性修改多個欄位的鍵名。
- **結構展平**：有助於在將嵌套 JSON 匯出至 CSV 等扁平格式前，重新命名欄位以維持結構清晰。

## 7. Remove Duplicates 節點

自動檢測並移除重複的資料項。

- **唯一鍵 (Unique Key)**：指定一個或多個欄位作為判斷重複的依據（例如 `email` 或 `order_id`）。
- **保留策略**：可以選擇保留「第一筆」或「最後一筆」出現的重複項。

---

📖 **進階技巧**：

- **表達式優化**：儘量在 `Edit Fields` 中使用表達式進行簡單轉換，減少對 `Code` 節點的依賴，以提高工作流的可讀性。
- **資料預覽**：在開發過程中，利用 n8n 的「Test Node」功能即時查看處理後的 JSON 結構。
